# 0011: Presence は Redis の TTL 付きキーで管理する

- ステータス: 採用
- 日付: 2026-07-11

## 背景

オンライン状態は頻繁に変わる一時情報であり、正確さより可用性と低コストが重要。
CLAUDE.local.md の制約: 切断イベントだけに依存しない / 複数端末を個別管理 /
Presence 障害がメッセージ送受信へ影響しない。

## 決定

- 保存先は Redis のみ(PostgreSQL に正本を置かない):
  - `presence:conn:{connectionId}` → userId、TTL 60秒
  - `presence:user:{userId}` → 接続 ID の Set、TTL 120秒
- SignalR 接続時に登録し、クライアントの heartbeat(30秒間隔の Hub メソッド呼び出し)で TTL を更新する。
  切断イベントを取りこぼしても TTL で自然にオフラインになる
- 1つ以上の生きた接続キーがあれば Online。端末ごとに接続キーを分ける
- 参照はワークスペースメンバーのみに許可し、クライアントは30秒間隔のポーリングで取得する。
  リアルタイム配信(presenceChanged イベント)は行わない — 大規模グループでの
  イベント増幅を避けるため(ADR 0007)。必要になれば人数閾値付きで導入する
- `IPresenceTracker` は例外を送出しない。Redis 障害時は「全員オフライン」に縮退し、
  メッセージ送受信・通知作成は継続する(通知は全員オフライン扱い → Push が増える方向に倒れる)
